# Copilot Expansion Guide

This file instructs code generation tools on how to expand the Oraphine repository. Follow these guidelines to maintain cohesion.

- **Respect the visual DNA:** reuse stems, chambers, wedges, crescents, forks, nodes, bars and halos.
- **Never imitate real-world scripts:** do not generate Arabic, Roman, runic or other letters.
- **Add new glyphs by mutation:** combine existing parts in new ways instead of inventing unrelated shapes.
- **Document everything:** update both markdown documentation and JSON inventories when adding new content.
- **Keep data and docs in sync:** every glyph entry in `data/glyphs/` should have a corresponding description in `docs/02_glyph_families/`.
- **Use templates:** when creating files, copy the appropriate template from `templates/`.
