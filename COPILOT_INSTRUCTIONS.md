# Copilot Instructions

This repository defines an invented alien glyph language called Oraphine. Your role is to expand the system without breaking visual cohesion.

## Core principles

- **Reuse existing structural motifs** (stems, chambers, crescents, wedges, forks, nodes, bars, halos) before inventing new ones.
- **Preserve asymmetrical near‑symmetry** and the tall/narrow proportions of the glyphs.
- **Avoid direct imitation of real‑world writing systems** such as Arabic, runes, kana, hieroglyphs or Latin letters.
- **Favour consistency over novelty:** expand by mutation and recombination of established parts rather than by random invention.

## When adding new glyph files

- Assign a unique ID and class (common, elevated, sigil, divider, number‑like, modifier).
- List the anchor, components and provide a concrete visual description.
- Include any semantic biases and note possible variants.
- Ensure the glyph clearly belongs to the existing family.

## When updating documentation

- Keep language concrete and visual. Explain how the system behaves in art and prompt generation.
- Add examples wherever possible.

## When adding prompt modules

- Emphasize coherence and repeated motifs.
- Include anti‑randomness constraints.
- Describe deployment modes such as text bands, columns, circular seals, relic markings and interface clusters.

## When adding examples

- Reuse the same script family across all examples.
- Demonstrate multiple deployment modes.
- Show both ceremonial and functional applications.

## Avoid

- Real‑world script imitation.
- Decorative icon packs with no family resemblance.
- Random novelty shapes that break the internal logic of Oraphine.
